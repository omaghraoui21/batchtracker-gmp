# Contributing

Thank you for considering a contribution to BatchTracker GMP.

This project welcomes practical contributions from production managers, pharmacists, QA professionals, industrial engineers, and software developers.

## Useful contributions

- Improve documentation
- Add production workflow examples
- Improve mobile UI screens
- Add batch status logic
- Add tests
- Suggest GMP-friendly fields or review steps
- Improve accessibility and usability for shop-floor users

## Principles

- Keep workflows simple enough for real production teams.
- Make calculations and status logic transparent.
- Avoid storing confidential production, employee, supplier, patient, or batch-sensitive data.
- Mark AI-generated text as draft decision support.
- Do not present the app as validated GMP software.

## Pull request checklist

- Explain the operational problem being solved.
- Keep changes small and reviewable.
- Update documentation when behavior changes.
- Avoid adding sensitive real production data.
- Add screenshots for visible UI changes when useful.

## Maintainer note

This project is maintained in public as part of a long-term learning and open-source effort by a pharmaceutical production manager learning software engineering and AI-assisted development.
