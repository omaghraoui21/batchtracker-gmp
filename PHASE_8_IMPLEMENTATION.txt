4. SECURITY & COMPLIANCE
================================================

✓ Comprehensive Audit Logging (/lib/auditLog.ts)
  - All critical actions logged for GMP compliance:
    * Batch creation, status changes
    * Step assignments and completions
    * Electronic signatures (1st and 2nd)
    * Deviation creation and updates
    * User creation, updates, and deletions
  - Each log entry includes:
    * Action type (CREATE, UPDATE, DELETE, SIGN, etc.)
    * Module (BATCH, USER, STEP, DEVIATION, etc.)
    * Entity type and ID
    * User ID, name, and role
    * Detailed description
    * Old and new values (for updates)
    * Timestamp (ISO 8601)
    * Optional metadata
  - Retrievable via getAuditLog() for entity-specific history
  - Dashboard view via getRecentAuditLog() for system-wide activity

✓ Role-Based Access Control (RBAC) (/components/RBACGuard.tsx)
  - RBACGuard component for screen-level protection
  - useHasRole hook for conditional rendering
  - useIsAdmin and useIsManager utility hooks
  - RoleBased component for inline permission checks
  - Clear "Access Denied" screens with role information
  - Enforced on:
    * User Management (Admin only)
    * Workflow Configuration (Admin only)
    * Batch Creation (Admin/Manager)
    * Critical actions (signatures, approvals)

✓ Audit Trail Integration
  - Batch details page shows audit history
  - User actions tracked in real-time
  - Complete traceability for regulatory compliance
  - Expandable history view (default: 5 entries, expandable to 50+)
