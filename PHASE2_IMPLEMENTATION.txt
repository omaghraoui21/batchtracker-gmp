## Points Techniques Importants

1. **Intégrité des Données**
   - Tous les horodatages générés par le serveur (NOW())
   - Pas de dates client-side pour éviter les problèmes de timezone
   - Transactions pour les opérations multi-tables

2. **Performance**
   - Indexes sur toutes les foreign keys
   - Queries optimisées avec select spécifiques
   - Pagination prête (limit utilisé)

3. **Sécurité**
   - RLS activé sur toutes les tables
   - Authentification requise pour toutes les opérations
   - Pas d'exposition de service_role_key côté client

4. **TypeScript**
   - Types stricts générés depuis le schéma
   - Pas d'erreurs TypeScript
   - Intellisense complet

5. **Navigation**
   - Routing dynamique pour les lots ([id])
   - Navigation entre les écrans
   - Retour avec Stack.Screen
