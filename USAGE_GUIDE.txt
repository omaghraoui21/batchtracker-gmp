GUIDE D'UTILISATION - BATCHTRACKER GMP
======================================

## Démarrage

L'application est déjà démarrée et le serveur Metro est actif. Vous pouvez accéder à l'application via Expo Go ou votre navigateur web.

## Navigation dans l'Application

### 1. Tableau de Bord (Dashboard)
- **Accès** : Onglet "Tableau de Bord" (icône maison)
- **Fonctionnalités** :
  * Visualisation des KPIs en temps réel :
    - Latence moyenne de vérification
    - Déviations critiques ouvertes
    - Lots actifs
  * Liste des 3 lots les plus récents
  * Cliquer sur un lot pour voir les détails
  * Rafraîchir en tirant vers le bas (pull-to-refresh)

### 2. Liste des Lots (Batches)
- **Accès** : Onglet "Lots" (icône layers)
- **Fonctionnalités** :
  * Barre de recherche pour filtrer par numéro ou produit
  * Statistiques rapides : Actifs, Prioritaires, Ce mois
  * Liste complète de tous les lots
  * Badges visuels :
    - 🚩 Priorité élevée
    - ⚠️ Déviations critiques
  * Cliquer sur un lot pour voir les détails complets
  * Rafraîchir avec pull-to-refresh

### 3. Détail d'un Lot
- **Accès** : Cliquer sur un lot depuis le Dashboard ou la liste
- **Sections** :

  #### En-tête
  - Numéro du lot et nom du produit
  - Statut actuel (En production / Terminé)
  - Badge de priorité si élevée

  #### Alerte Déviations
  - Bannière rouge si déviations critiques ouvertes
  - Indique le blocage de validation

  #### Timeline des Étapes
  - Visualisation graphique de toutes les étapes
  - Pour chaque étape :
    * Indicateur visuel (✓ terminée, • en cours, ○ en attente)
    * Nom de l'étape
    * Rôle requis
    * Horodatages (début, fin)
    * SLA avec badge "Hors SLA" si dépassé
    * Statut actuel

  #### Liste des Déviations
  - Sévérité (Critique / Majeure / Mineure)
  - Titre et description
  - Statut (Ouvert / Résolu)
  - Date de signalement

  #### Actions Disponibles
  - **Marquer comme Prêt** : Valider l'étape actuelle
    * Désactivé si déviations critiques ouvertes
    * Passe automatiquement à l'étape suivante
    * Enregistre dans l'historique
  - **Signaler une Déviation** : Créer une nouvelle déviation

## Configuration des Workflows

### Accéder à la Configuration
1. Onglet "Plus"
2. Section "Gestion"
3. Cliquer sur "Workflows"

### Sélectionner un Modèle
- Les modèles disponibles s'affichent en haut
- Cliquer sur un modèle pour voir ses étapes
- Badge indique le nombre d'étapes

### Gérer les Étapes
- Chaque étape affiche :
  * Numéro d'ordre
  * Nom
  * Rôle requis
  * SLA en heures

#### Ajouter une Étape
1. Cliquer sur le bouton "+" bleu
2. Remplir le formulaire :
   - Nom de l'étape
   - Rôle requis
   - SLA en heures
3. Cliquer sur "Enregistrer"

#### Modifier une Étape
1. Cliquer sur l'icône crayon (✏️)
2. Modifier les champs
3. Cliquer sur "Enregistrer"

#### Réorganiser les Étapes
- Cliquer sur ↑ pour monter
- Cliquer sur ↓ pour descendre
- L'ordre est sauvegardé automatiquement

#### Supprimer une Étape
1. Cliquer sur l'icône poubelle (🗑️)
2. Confirmer la suppression
3. L'étape est supprimée définitivement

## Workflow de Validation d'un Lot

### Étapes Typiques
1. **Production** (Opérateur, SLA: 4h)
   - Fabrication du lot
   - Upload des documents requis

2. **Superviseur** (Vérificateur, SLA: 2h)
   - Vérification de la production
   - Validation ou escalade

3. **QA** (Qualité, SLA: 8h)
   - Contrôle qualité final
   - Tests et analyses
   - Validation finale

### Processus de Validation
1. Accéder au détail du lot
2. Vérifier que l'étape actuelle est "En cours"
3. Vérifier qu'il n'y a pas de déviations critiques
4. Cliquer sur "Marquer comme Prêt"
5. Confirmer l'action
6. Le lot passe automatiquement à l'étape suivante

### Gestion des Blocages
Si validation bloquée :
- Vérifier les déviations critiques
- Résoudre ou escalader les déviations
- Une fois résolues, la validation sera possible

## Indicateurs Visuels

### Couleurs des Statuts
- 🟢 Vert : Terminé / Succès
- 🔵 Bleu : En cours / Actif
- 🟡 Jaune : En attente
- 🔴 Rouge : Critique / Hors SLA / Bloqué
- ⚫ Gris : Annulé / Inactif

### Badges SLA
- "Hors SLA" en rouge : Étape dépassant le délai alloué
- Horloge rouge : Deadline dépassée

### Badges de Priorité
- 🚩 Rouge : Priorité élevée
- Pas de badge : Priorité normale

### Badges de Déviation
- ⚠️ Rouge : Déviations critiques ouvertes
- Compte le nombre de déviations

## Conseils d'Utilisation

### Performance
- Utilisez le pull-to-refresh pour actualiser les données
- Les données sont mises en cache pour réduire les appels API
- Les transitions sont enregistrées en temps réel

### Workflow Optimal
1. Commencer la journée : Rafraîchir le Dashboard
2. Identifier les lots avec SLA risqué
3. Traiter en priorité les lots "Hors SLA"
4. Valider les étapes dès que possible
5. Signaler immédiatement les déviations

### Bonnes Pratiques
- Toujours vérifier les documents avant validation
- Signaler les déviations dès leur découverte
- Maintenir les workflows à jour
- Former les équipes aux rôles et responsabilités
